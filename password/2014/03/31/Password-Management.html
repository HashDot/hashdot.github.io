Password Management

Mar 31, 2014

Immer dabei!

Mit der hier vorgestellten Lösung habt Ihr immer alle Passwörter dabei und das Sicher und Aktuell! Es gibt eine große Anzahl an Software die uns dabei Helfen unsere Passwörter sicher zu verwalten. Doch warum der ganze Aufwand?

Immer wieder werden die Datenbanken von diversen Websites oder Online Diensten gehackt und entwenden so eine menge Daten. Viele von euch nutzen auf jeder Platform das gleiche Passwort und das ist ein großer Fehler. Hat man erstmal das Passwort geklaut und entschlüsselt, oder es liegt sogar in Klartext vor, kann man schnell all eure Dienste kom­pro­mit­tie­ren.

Aus diesem Grund sollte man wirklich für jeden Dienst ein eigenes und gutes Passwort verwenden. Doch niemand kann sich alle merken. Abhilfe schafft eine gute Passwort Verwaltung. Populär ist unter OSX 1Password, gibt es auch für Windows, Android und iOS. Das ist eine tolle Passwort Verwaltung und funktioniert gut, doch der Preis ist nicht ohne. Da kommen für den Mac plus iPhone mal locker 60€ zusammen.

OpenSource schafft Alternative

Auf dem OpenSource Markt der Passwort Verwaltungen ist mir KeePass sehr positiv aufgefallen. Den Client gibts es für jede Plattform wenn auch nicht vom selben Entwickler aber das ist das schöne an OpenSource. Doch was bietet uns der Vorteil OpenSource noch? Nicht nur Ich sondern alle IT-Security Experten raten Sicherheitsrelevante Software als OpenSource zu nutzen. Nur so können wir uns sicher sein, das im Quellcode keine Backdoors oder ähnliches versteckt sind. Denn was wäre eine Passwort Verwaltung mit einer Backdoor? Wir könnten dann auch unsere ganzen Passwörter auf ein Blatt Papier schreiben und aufhängen.

Zentral/Dezentrale Verwaltung der Passwörter

KeePass Verwaltet unsere Passwörter in einer eigenen Datenbank. Diese können wir ablegen wo immer wir es für richtig halten. Aus diesem Grund können wir diese Zentral oder auch Dezentral lagern. Für eine Dezentrale Lagerung müssen wir uns allerdings eine synchronisations Möglichkeit überlegen. In diesem Beispiel bleiben wir bei der Zentralen Verwaltung.

Da die Datenbank mit unseren Passwörtern Verschlüsselt ist brauchen wir uns keine Gedanken darüber zu machen ob Google & Co. diese lesen kann. Aus diesem Grund habe ich meine Datenbank auf meinem Google Drive liegen. Somit kann ich auf diese von jedem PC und Smartphone aus zugreifen.

Neue Passwörter …

Fangen wir nun damit an unseren Online Diensten neuen Passwörter zuzuordnen. Wir können in KeePass Kategorien anlegen unter diesen wir dann unsere Einträge speichern.

Nun könnt Ihr Stück für Stück eure Passwörter in KeePass eintragen bzw. diese von KeePass sogar erstellen lassen. Dafür gibt es genug Optionen die eure Passwörter mehr als sicher machen. Bisher hatte ich auch noch auf keiner Plattform ein Problem mit diesen zufallsgenerierten Passwörter.

__ ACHTUNG! __ Wenn Ihr euer Passwort von eurer KeePass Datenbank verliert habt Ihr kein Zugriff mehr auf eure Passwörter!

Multi User Support

In meinem Job gibt es eine Menge an Passwörtern. Diese werden in der gleichen Abteilung meist von mehreren Usern gleichzeitig genutzt. KeePass gibt uns die Möglichkeit das viele User die gleiche Datenbank nutzen können.

Die KeePass 1.x Version unterstüzt nur Read-Only oder Normal Modus. Im Normal Modus bekommt dann der User die Rechte über die Datei.

Die KeePass 2.x Version besitzt schon ein paar mehr Funktionen. Hier gibt es die Synchronisations Methode. Damit können alle User die gleiche Datenbank nutzen und diese auch verändern. Alle Benutzer bekommen dann automatisch ohne neuladen der Datenbank die aktualisierten Daten.

Möchte man bestimmten Usern nur das Lesen ermöglichen muss das über das jeweilige Dateisystem geschehen da KeePass keine Benutzerverwaltung besitzt.

URL Feld

Das URL Feld ist mehr als es den anschein macht. Es unterstüzt nicht nur http:// sondern viele Formate wie ftp:// mailto:// ssh:// cmd:// usw. Das ermöglicht uns mit Platzhaltern zu arbeiten und mit einem Klick eine SSH Verbindung per Putty aufzubauen oder eine neue RDP Session zu starten. KeePass verknüpft die Pfade mit dem Standardprogramm was dafür konfiguriert ist.

Möchten wir uns z.B. per API bei einem Dienst Anmelden könnte das wie folgt aussehen:

http://example.com/login/{USERNAME}/{PASSWORD}

Es gibt eine Reihe von Platzhaltern die wie Verwenden können. Zu finden sind diese hier: KeePass Placeholder

TAN Support

Eine sehr tolle Möglichkeit bietet uns der TAN Support. Für alle die noch keine SMS-TAN für Ihre Bank nutzen, sondern eine TAN Liste besitzen, ist das toll. Ihr könnt hier alle TAN Nummern eintragen und habt Sie so immer verschlüsselt bei euch. Benutzt Ihr eine TAN wird diese automatisch als benutzt markiert. Aber nicht nur für die Bank ist das interessant. Ich nutze das für meine Uni da wir dort für Prüfungsanmeldung usw. auch ein TAN System benutzen.

Datenbank reparieren

Wir wollen zwar nicht hoffen das dieser Fall mal eintreten sollte das unsere Datenbank defekt ist aber KeePass stellt uns eine Reparatur Hilfe zu Verfügung. Diese findet Ihr bei KeePass 1.x unter: Tools -> Repair KeePass Database File … und unter KeePass 2.x: Erstellt Ihr eine neue Datenbank und Importert die alte Datenbank im KeePass 2.x Repair Mode.

Eine defekte Datenbank kann dann auftreten wenn etwas beim Speichern oder Übertragen falsch gelaufen ist oder die Festplatte defekt ist. KeePass Überprüft aber alle Summen direkt nach bzw. bei dem Speichervorgang. Aus diesem Grund ist KeePass ziemlich sicher.

Clients

KeePass ist OpenSource und aus diesem Grund gibt es eine große Anzahl an Clients für fast jede Plattform. Unter anderem für:

  • iOS
  • Android
  • OSX
  • Windows
  • Linux
